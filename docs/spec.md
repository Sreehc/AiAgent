# AiAgent 前端 UI 升级功能规格

> 来源：`docs/requirements.md`。本文档是面向设计、开发和测试的执行规格，范围仅覆盖前端 UI 与体验升级；不重写后端业务能力，不迁移 Next.js，不接入 assistant-ui runtime。

## 1. 功能概述

本次升级目标是把 AiAgent 从“功能完整的后台工具”提升为“专业、可信、好看的 AI 研究工作台”。核心策略是先完成高保真设计稿，再按现有 React Router + Vite + Tailwind + Radix-style 组件体系增量实现。

### 1.1 范围

| 模块 | 规格 |
| --- | --- |
| 视觉系统 | 采用 graphite/zinc 中性色 + restrained cyan/teal 强调色；亮/暗主题配对；默认跟随系统 |
| 字体 | 推荐 Geist Sans 作为 UI 字体，IBM Plex Mono 作为代码、ID、日志、数字字体 |
| 通用组件 | 统一 Panel、Button、Badge、StatusPill、Table、Tabs、Alert、EmptyState、Skeleton、Dialog、DropdownMenu、CommandPalette |
| 研究工作台 | 左侧会话、中间 Agent feed、右侧 Inspector Tabs |
| 知识库 | RAG cockpit：知识库列表、文档索引状态、检索测试、证据卡片 |
| 图片工作室 | 强化预览、历史缩略图、批量选择、收藏、对比视图、下载集合 |
| 管理端 | 新增仪表盘式总览，聚合模型、MCP、审计、RAG 评估和风险状态 |
| 审计/RAG | 结构化表格 + 展开详情，默认不直接展示原始 JSON |
| 回归验证 | 纳入 Playwright 截图回归或端到端检查 |

### 1.2 不在范围内

- 不重写 Spring Boot 后端业务逻辑。
- 不迁移到 Next.js App Router。
- 不使用 `npx assistant-ui init --yes`。
- 不把系统改造成单一聊天应用。
- 不移除 RAG、MCP、图片生成、历史回放、账号中心、管理员能力。
- 不引入 assistant-ui runtime；assistant-ui 只作为交互参考。

### 1.3 仍需确认的问题

| 问题 | 推荐默认值 | 影响 |
| --- | --- | --- |
| 字体授权和托管方式 | Geist Sans + IBM Plex Mono，本地托管优先，无法托管时使用系统 fallback | 影响首屏加载、字体一致性和部署资产 |
| 高保真设计稿覆盖范围 | 第一轮覆盖核心五页：登录/注册、研究工作台、知识库、图片工作室、管理总览；第二轮覆盖历史、账号、模型、MCP、审计、RAG、404 | 影响设计排期和开发开工顺序 |
| Playwright 截图基线 | 桌面 1440x900、笔记本 1280x800、平板 768x1024、手机 390x844；亮色、暗色各一组 | 影响 CI 时长和截图基线维护 |
| 图片收藏持久化 | 首版优先使用前端本地持久化；若要求跨设备同步，需要后端接口确认 | 影响图片工作室实现边界 |

## 2. 用户流程

### 2.1 设计到实现流程

```text
requirements.md / spec.md
  ↓
高保真设计稿
  ↓
产品 + 设计 + 技术评审
  ↓
视觉 token 与组件升级
  ↓
核心页面实现
  ↓
Playwright 截图/端到端回归
  ↓
交付验收
```

规则：

- 未完成高保真评审前，不进入大规模页面实现。
- token 和通用组件优先于页面级改造。
- 每一阶段完成后必须能回归现有核心业务流程。

### 2.2 研究任务流程

```text
进入研究工作台
  ↓
选择/新建会话
  ↓
输入研究问题，选择执行模式、策略、知识库和复用产物
  ↓
提交任务
  ↓
Agent feed 追加计划、工具调用、证据、产物和报告事件
  ↓
右侧 Inspector Tabs 查看产物、证据、记忆、工具详情
  ↓
完成后复用产物或进入历史回放
```

异常路径：

- SSE 中断：保留已收到事件，提示可从历史恢复。
- 任务失败：显示失败状态、错误摘要和可恢复入口。
- 无会话：禁用运行按钮，提示先创建或选择会话。

### 2.3 知识库流程

```text
进入知识库
  ↓
选择/创建知识库
  ↓
上传文档
  ↓
查看解析和索引状态
  ↓
执行检索测试
  ↓
查看证据卡片和召回质量
  ↓
返回研究工作台绑定知识库
```

### 2.4 图片工作室流程

```text
进入图片工作室
  ↓
选择文本生图或参考图编辑
  ↓
输入 Prompt，选择尺寸和会话
  ↓
生成/编辑图片
  ↓
查看最新输出和历史缩略图
  ↓
批量选择、收藏、对比或下载集合
```

### 2.5 管理端流程

```text
管理员进入管理总览
  ↓
查看模型、MCP、任务失败、登录失败、RAG 评估风险
  ↓
点击指标跳转到对应配置或审计页面
  ↓
处理模型、MCP、邀请码、审计或 RAG 评估事项
```

## 3. 功能规则

### 3.1 视觉规则

- 主视觉方向固定为 graphite/zinc + cyan/teal。
- cyan/teal 只用于主操作、active navigation、focus、运行中状态和关键强调，不大面积铺满页面。
- amber 只用于 warning，red 只用于 error/destructive，green 只用于 success。
- 亮色和暗色主题必须成对设计，不能只从一个主题反推另一个主题。
- 用户未设置主题时跟随系统；用户手动切换后持久化主题偏好。

### 3.2 字体规则

- 推荐 UI 字体：Geist Sans。
- 推荐代码/数字/ID 字体：IBM Plex Mono。
- 动态数字、计数、耗时、分数、token 数应使用 tabular numbers。
- 长 ID 默认截断显示，hover/focus 或展开详情中可查看完整值。

### 3.3 通用组件规则

- `Button` 必须有明确 loading、disabled、hover、focus、active 状态。
- `IconButton` 必须提供 `aria-label`，可见点击区域不小于 40x40px。
- `Badge` 默认使用紧凑矩形或轻微圆角，不再全局使用胶囊风格。
- `StatusPill` 只表达状态，不承担普通装饰用途。
- `Panel` 用于有明确边界的工具区、重复项目、模态内容；避免页面区块全都变成卡片。
- `Table` 默认支持横向溢出处理，数字列右对齐，状态列保持紧凑。
- 原始 JSON 默认折叠，摘要优先；展开后使用 monospace。

### 3.4 工作台规则

- 中间区域为主工作流，优先展示任务输入、Agent feed 和最终报告。
- 右侧 Inspector Tabs 至少包含：产物、证据、记忆、工具。
- 工具调用默认展示工具名、状态、耗时、摘要；请求/响应 payload 放入展开详情。
- 证据默认展示来源、分数、citation、section、内容预览。
- 暂停、继续、取消、失败、完成状态必须视觉上可区分。

### 3.5 图片工作室规则

- 最新输出区域为视觉主区域。
- 历史记录使用缩略图网格，图片边缘需要清晰 outline 或阴影。
- 批量选择模式下，图片卡片必须显示选中状态和选择计数。
- 对比视图首版支持 2 到 4 张图片并排对比。
- 下载集合应以当前选中图片为输入；没有选中项时禁用。
- 收藏首版推荐本地持久化；若后续要求跨设备同步，应补充后端接口。

### 3.6 管理总览规则

- 总览页只聚合关键风险和状态，不替代详细配置页。
- 每个指标卡都应可点击跳转到对应页面或筛选后的列表。
- 异常 provider、MCP unhealthy、近期失败任务、登录失败、RAG 评估失败应优先展示。

## 4. 页面和交互

### 4.1 路由与页面

| 路由 | 页面 | 主要交互 |
| --- | --- | --- |
| `/login` | 登录 | 登录、跳转注册、找回密码 |
| `/register/invite` | 邀请注册 | 输入邀请码、创建账号、返回登录 |
| `/forgot-password` | 找回密码 | 发起找回、跳转重置 |
| `/reset-password` | 重置密码 | 输入 token、新密码、确认密码 |
| `/workspace/chat` | 研究工作台 | 创建会话、运行任务、暂停/继续/取消、查看 Inspector |
| `/workspace/knowledge-bases` | 知识库 | 创建 KB、上传文档、索引、检索测试、版本管理 |
| `/workspace/image-generation` | 图片工作室 | 生成/编辑图片、批量选择、收藏、对比、下载集合 |
| `/workspace/history` | 历史回放 | 筛选会话、查看回放、复用产物 |
| `/account` | 账号中心 | 资料、密码、API 配置、登录日志 |
| `/admin/settings` | 模型配置 | 新增/编辑/启停/测试模型、邀请码 |
| `/admin/mcp-servers` | MCP 服务器 | 注册、发现工具、健康检查、测试工具 |
| `/admin/audit` | 审计 | 筛选、结构化表格、展开详情 |
| `/admin/rag-evaluations` | RAG 评估 | 用例管理、运行评估、指标详情 |
| `/admin/overview` | 管理总览 | 查看风险指标并跳转处理 |
| `*` | 404 | 返回工作台或登录 |

`/admin/overview` 为新增建议路由；如果团队不希望增加路由，可作为 `/admin/settings` 上方的总览区，但测试用例仍按“管理总览”能力验收。

### 4.2 响应式交互

- 桌面端优先使用侧栏 + 主内容 + 右侧 Inspector。
- 小于 1200px 时，工作台右侧 Inspector 可移到主内容下方或变为抽屉。
- 小于 900px 时，主导航进入移动菜单，页面布局改为单列。
- 手机端不得依赖 hover 展示关键信息。

### 4.3 高保真设计稿交互要求

第一轮设计稿建议覆盖：

- 登录/注册组合页。
- 研究工作台完整状态：空态、运行中、完成、失败。
- 知识库：文档列表、索引状态、检索结果。
- 图片工作室：最新输出、历史、批量选择、对比视图。
- 管理总览：正常状态、风险状态。

第二轮设计稿覆盖：

- 历史回放、账号中心、模型配置、MCP、审计、RAG 评估、404。

## 6. 状态变化

### 6.1 主题状态

```text
system
  ├── prefers light → light
  └── prefers dark → dark

user override
  ├── set light → persist light
  └── set dark → persist dark
```

### 6.2 研究任务状态

| 状态 | UI 表现 | 可用操作 |
| --- | --- | --- |
| 未选择会话 | 空态提示 | 创建/选择会话 |
| IDLE | 可编辑任务 | 运行研究、绑定知识库 |
| PENDING/RUNNING | Agent feed 追加事件 | 暂停、取消 |
| PAUSED | feed 保留当前位置 | 继续、取消 |
| CANCEL_REQUESTED | 显示取消中 | 等待状态收敛 |
| CANCELLED | 显示已取消 | 可发起新任务 |
| COMPLETED | 展示报告、证据、产物 | 复用产物、历史回放 |
| FAILED/TIMED_OUT | 展示错误摘要 | 查看详情、历史恢复、重新运行 |

### 6.3 图片选择状态

| 状态 | 规则 |
| --- | --- |
| 普通浏览 | 点击图片打开详情或预览 |
| 批量选择 | 卡片显示 checkbox 或 selection overlay |
| 已选择 | 显示选中边框和选择序号 |
| 已收藏 | 显示收藏标记；首版推荐本地持久化 |
| 对比中 | 2 到 4 张图片进入对比视图 |
| 下载中 | 下载按钮 loading，成功后 toast 提示 |

### 6.4 表格状态

所有表格至少支持：

- Loading：骨架或占位行。
- Empty：清晰空态和下一步动作。
- Error：错误摘要和重试入口。
- Filtered：显示当前筛选条件。
- Expanded：查看详情或原始 payload。

## 7. 权限规则

- 未登录用户只能访问登录、邀请注册、找回密码、重置密码和 404。
- 登录用户可访问工作台、知识库、图片、历史、账号中心。
- 管理员可访问模型配置、MCP、审计、RAG 评估、管理总览。
- 普通用户不显示管理导航；直接访问管理路由时展示无权限页面。
- UI 层不得展示用户无权访问的会话、知识库、产物或审计数据。
- 前端权限只做展示和交互控制，最终权限以服务端接口为准。

## 8. 异常处理

| 异常 | 处理 |
| --- | --- |
| API 401/403 | 清晰提示登录失效或无权限；不得显示空白页 |
| API 500 | 页面级 Alert 展示错误摘要，保留当前输入 |
| SSE 中断 | 保留已收到事件，提示可从历史回放恢复 |
| 图片加载失败 | 显示图片占位、失败原因和重试/打开原图入口 |
| 字体加载失败 | 自动回退系统字体，不阻塞页面渲染 |
| metrics 解析失败 | 展示原始 metrics 展开详情，并提示结构化解析失败 |
| JSON payload 过长 | 默认折叠，设置最大高度和滚动 |
| 移动端空间不足 | 次级面板折叠或进入 tabs/drawer，不允许内容重叠 |
| Playwright 截图差异 | 标记差异页面、主题、viewport；需人工确认后更新基线 |

## 9. 边界情况

- 没有会话：工作台显示创建会话入口，运行按钮禁用。
- 没有知识库：研究任务仍可运行，知识库区域显示普通上下文提示。
- 文档索引失败：文档行显示失败状态和错误摘要，支持重试索引。
- 检索无命中：展示空态和调整查询建议。
- 没有图片历史：图片工作室展示生成引导，不显示空白画廊。
- 批量下载无选择：下载集合按钮禁用。
- 对比图片少于 2 张：对比按钮禁用并提示至少选择 2 张。
- 对比图片超过 4 张：只允许进入前 4 张或提示减少选择，具体设计稿确认。
- 超长标题、文件名、ID、URL：默认截断，详情中完整展示。
- 小屏幕下表格列过多：优先改为卡片行或横向滚动，不压缩到不可读。
- 暗色主题下状态色不得只靠颜色表达，需配合图标、文字或形状。
- 管理总览无风险项：展示正常状态摘要，不制造警告噪音。

## 10. 验收标准

### 10.1 设计验收

- 高保真设计稿覆盖第一轮核心五页。
- 设计稿包含亮色和暗色主题。
- 设计稿包含空态、加载态、错误态、运行中、完成、失败状态。
- 视觉方向符合 graphite/zinc + cyan/teal，不回到绿橙主色。
- 字体方案、token、组件状态在设计稿中明确标注。

### 10.2 开发验收

- 现有路由和核心业务流程无回归。
- 主题默认跟随系统，手动切换可持久化。
- 通用组件状态统一，Button、Badge、StatusPill、Table、Panel 行为符合规则。
- 工作台改为 Agent feed + Inspector Tabs 信息架构。
- 知识库检索结果以证据卡片展示。
- 图片工作室支持批量选择、收藏、对比视图和下载集合。
- 管理端提供总览能力，并能跳转到现有管理模块。
- 审计与 RAG 评估默认结构化展示，原始数据只作为展开详情。

### 10.3 测试验收

- `frontend` 构建通过。
- 核心路径通过手动验证：登录、运行研究、知识库检索、图片生成、历史回放、模型配置、MCP 健康检查、审计筛选、RAG 评估。
- Playwright 覆盖至少以下 viewport：1440x900、1280x800、768x1024、390x844。
- Playwright 覆盖亮色和暗色主题。
- 截图回归覆盖至少：登录、研究工作台、知识库、图片工作室、管理总览。
- E2E 或交互测试覆盖：主题切换、Inspector Tabs、图片批量选择/对比、审计展开详情、RAG metrics 展开详情。
- 移动端不得出现文本重叠、横向不可控滚动、按钮文本溢出或关键操作不可达。
