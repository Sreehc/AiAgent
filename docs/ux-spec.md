# UX 规格：AiAgent V1

> 文档状态：基于 2026-06-13 前端路由、页面和后端接口复核。本文档描述当前可实现 UX；尚无后端支撑的交互会明确标注。

## 1. Overview

AiAgent V1 是一个登录后使用的多智能体工作台，用户的核心目标是安全登录系统、发起研究任务、查看结构化结果、管理知识库、生成图片并回看历史。管理员在此基础上拥有配置模型、MCP 和邀请码的能力。

## 2. Information Architecture

### Navigation Structure

```text
AiAgent
├── 登录
├── 邀请注册
├── 找回密码
├── 重置密码
├── 工作台
│   ├── 聊天研究
│   ├── 知识库
│   ├── 图片生成
│   └── 历史会话
├── 账号中心
│   ├── 个人资料
│   ├── 修改密码
│   └── 登录日志
└── 基础配置
    ├── 模型配置
    ├── MCP 配置
    ├── RAG 评估
    ├── 系统审计
    └── 邀请码管理
```

### Key User Paths

1. 登录使用：登录页 → 工作台 → 发起研究 → 查看结果
2. 邀请注册：邀请链接 → 注册页 → 登录页 → 工作台
3. 知识增强：工作台 → 知识库 → 上传文档 → 绑定会话 → 发起研究
4. 图片生成：工作台 → 图片生成 → 文本生图/参考图编辑 → 保存产物
5. 历史回放：工作台 → 历史会话 → 回放详情 → 打开产物

## 3. User Flows

### Flow 1: 邀请注册与登录

**Trigger:** 用户收到邀请码或邀请链接  
**User goal:** 注册账号并登录进入工作台

```text
邀请链接
  ↓
注册页
  ↓
填写账号信息
  ↓
注册成功
  ↓
登录页
  ↓
工作台
```

错误路径：

- 邀请码失效：展示错误页，提示联系管理员
- 账号名重复：停留注册页，字段级报错
- 登录失败：表单错误提示，不跳转
- 找回密码请求成功：展示统一成功提示；开发环境可使用日志 sender，SMTP 环境会发送重置链接，重置页支持 token 输入。

### Flow 2: 发起研究任务

**Trigger:** 用户在工作台输入研究问题  
**User goal:** 获取规划过程和研究报告

```text
工作台空态
  ↓
输入问题
  ↓
发送任务
  ↓
SSE 执行过程
  ↓
报告结果与产物
```

错误路径：

- SSE 中断：展示“连接中断，可查看历史回放”
- 执行失败：保留上下文并显示失败卡片

### Flow 3: 上传知识库并绑定会话

**Trigger:** 用户希望增强研究上下文  
**User goal:** 使用私有资料参与后续研究

```text
知识库页
  ↓
上传文档
  ↓
切分/索引中
  ↓
索引完成
  ↓
返回工作台绑定知识库
```

### Flow 4: 图片生成

**Trigger:** 用户需要研究配图或海报  
**User goal:** 生成可复用图片

```text
图片生成页
  ↓
输入提示词或上传参考图
  ↓
提交生成
  ↓
结果展示
  ↓
保存到产物
```

## 4. Screens

### Screen: 登录页

**URL/Route:** `/login`  
**Purpose:** 让用户进入系统  
**Entry points:** 未登录访问任意业务页、注册成功后跳转

#### Components

| Component | Description | Behavior |
|-----------|-------------|----------|
| Logo 区域 | 产品标识与一句话说明 | 固定展示 |
| 登录表单 | 用户名、密码 | 支持回车提交 |
| 登录按钮 | 主 CTA | 提交中显示 loading |
| 找回密码入口 | 跳转找回页 | 文本链接 |

#### States

| State | Appearance | Trigger |
|-------|------------|---------|
| Default | 表单空白 | 初始进入 |
| Loading | 按钮 loading | 提交中 |
| Error | 表单提示错误 | 登录失败 |

### Screen: 邀请注册页

**URL/Route:** `/register/invite`  
**Purpose:** 通过邀请码创建账号

#### Required Fields

- 邀请码
- 用户名
- 显示名称
- 密码
- 确认密码

### Screen: 聊天工作台

**URL/Route:** `/workspace/chat`  
**Purpose:** 发起研究任务并查看过程

#### Layout

```text
┌──────────────────────────────────────────────┐
│ 顶栏: Logo / 全局导航 / 用户菜单             │
├───────────────┬──────────────────────────────┤
│ 左侧会话列表   │ 中央对话与执行流             │
│               │ 输入框 / 计划 / 工具 / 结果   │
├───────────────┴──────────────────────────────┤
│ 底部输入区：问题输入 / 模式选择 / 发送按钮    │
└──────────────────────────────────────────────┘
```

#### Key Components

- 最近会话列表
- 当前对话流
- 计划步骤卡片
- 工具调用卡片
- 产物面板
- 知识库绑定入口
- 输入框与发送按钮

#### States

- 空态：展示示例提示词和开始指引
- 执行中：滚动追加事件流
- 暂停：显示继续入口，时间线保留当前步骤
- 取消：显示取消状态，不混同失败
- 完成：显示报告与产物入口
- 失败：显示失败状态与重试建议

当前边界：工作台支持运行级取消、暂停和继续；暂停/取消会在步骤边界或 provider 调用返回后收敛，不能强杀已发出的外部调用。

### Screen: 知识库工作区

**URL/Route:** `/workspace/knowledge-bases`  
**Purpose:** 管理用户私有知识库

#### Components

- 知识库列表
- 新建知识库按钮
- 文档上传面板
- 文档状态表格
- 检索测试输入框

### Screen: 图片生成工作区

**URL/Route:** `/workspace/image-generation`  
**Purpose:** 完成文本生图和参考图编辑

当前边界：文本生图链路已完成；参考图编辑在 `local-mock` 可验证，OpenAI-compatible provider 已实现 multipart 契约，真实可用性取决于外部 provider 兼容性。

#### Components

- 模式切换：文本生图 / 参考图编辑
- Prompt 输入框
- 上传参考图控件
- 生成按钮
- 结果画廊
- 历史记录面板

### Screen: 历史会话页

**URL/Route:** `/workspace/history`  
**Purpose:** 查看历史执行和回放

#### Components

- 会话筛选栏
- 会话列表
- 回放详情抽屉或详情页
- 计划时间线
- 工具调用列表
- 产物引用列表

### Screen: 账号中心

**URL/Route:** `/account`  
**Purpose:** 管理个人信息和账号安全

#### Sub-sections

- 个人资料
- 修改密码
- 登录日志
- 退出登录

### Screen: 基础配置页

**URL/Route:** `/admin/settings`  
**Purpose:** 管理模型、MCP、邀请码

#### Visibility

- 管理员可见
- 普通用户不可见

当前边界：

- 模型配置支持新增、编辑、启停、删除、默认和连接测试。
- MCP 配置支持新增、编辑、删除、发现、健康检查和工具测试；健康检查会真实探测服务并展示延迟、工具数和错误分类。
- 管理端提供系统审计与 RAG 评估页面。

## 5. Interaction Specs

### Interaction: 发送研究任务

- Trigger：点击发送按钮或回车
- Response：立即插入用户消息，占位“正在规划/执行”
- Feedback：输入区进入 loading 并展示执行过程；运行中可暂停、继续或取消，断流后可从详情/历史恢复。

### Interaction: 绑定知识库

- Trigger：在工作台选择知识库
- Response：绑定状态立即显示在当前会话头部
- Feedback：失败时 toast 提示

### Interaction: 图片生成

- Trigger：点击生成
- Response：显示任务卡片和加载占位
- Feedback：生成完成后在结果区追加图片

### Interaction: 查看历史回放

- Trigger：点击历史会话
- Response：展开回放详情
- Feedback：先显示骨架屏，再加载计划、工具、产物

## 6. Empty / Loading / Error

### Empty States

- 工作台无会话：展示示例研究问题
- 知识库为空：展示上传引导
- 图片历史为空：展示首次生成引导
- 登录日志为空：展示“暂无登录记录”

### Loading States

- 登录按钮 loading
- 会话列表骨架屏
- SSE 执行中动态流
- 历史回放详情骨架屏

### Error States

- 登录失败：表单内错误提示
- 邀请失效：单独错误页
- 知识库上传失败：文件级错误提示
- SSE 断连：页面横幅提示

## 7. Responsive Behavior

- Desktop：主体验形态，三栏或双栏布局
- Tablet：侧栏可折叠，结果区优先
- Mobile：仅保证基础可用，聊天、历史、账号中心单栏显示

## 8. UX Rules

- 所有长任务必须有可见执行反馈
- 所有用户私有资源页必须有明确归属提示
- 管理员能力与普通用户能力在导航和页面级都要隔离
- 历史与产物的入口必须稳定可见，避免结果生成后丢失

### 8.1 页面模式

| 分类 | 页面 | 模式 |
| --- | --- | --- |
| Auth | 登录、注册、找回密码、重置密码 | 双栏认证表单，移动端单列 |
| Work | 研究工作台 | 三栏/双栏执行工作区 |
| Knowledge | 知识库 | 资源列表 + 详情/文档/检索测试 |
| Studio | 图片工作室 | 参数面板 + 结果画廊 + 历史 |
| Observability | 历史回放 | 过滤列表 + 执行详情 + 产物 |
| System | 模型配置、MCP 服务器、系统审计、RAG 评估 | 配置资源管理页 |
| Account | 账号中心 | 分区设置页 |

### 8.2 视觉系统约束

风格：浅色、克制、信息密度较高的 AI operations console；视觉质量来自清晰层级、稳定布局、低噪声色彩和准确状态表达。

禁止：

- 大面积紫蓝渐变背景
- 毛玻璃 / backdrop filter 装饰
- 装饰性光斑、漂浮粒子
- 超大圆角卡片堆叠
- 营销页式 hero 区
- 每页独立色彩主题

设计 token 必须涵盖 color、typography、spacing（4px 基础网格）、radius（6/8/12/16/pill）、shadow（仅 dialog/popover/toast/command palette）、z-index（shell/overlay/modal/toast）。

### 8.3 组件状态

通用组件必须覆盖 default、hover、active、focus、disabled、loading、error 状态；status 颜色不得作为唯一信息来源，须配合文本或图标。

## 9. 当前产品能力边界

- 会话历史可回放，历史产物可作为新任务上下文复用。
- 产物面板支持报告、图片、参考图、附件、工具输出和上下文片段。
- 知识库支持创建、更新、整体删除、上传、索引、重建索引、检索测试、单文档删除、预览、下载、版本列表和版本恢复。
- 个人 API 配置可维护、可测试，并会影响 Chat、Embedding、Image 实际模型运行。
- 管理员可查看系统级用户、失败任务、工具调用和登录审计。
- 图片任务仍是同步 provider 调用，独立的图片任务暂停/取消不是当前交互闭环。
